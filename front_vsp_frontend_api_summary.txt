前端 VSP/AIP 接口对接说明

一、本次前端对接的 Alpha 后端接口

1. 机场数据
前端方法：
vspAPI.airports(icaoCode?: string)

后端接口：
GET /api/v1/vsp/airports
GET /api/v1/vsp/airports?icao_code=VHHH

返回类型：
VspAirport[]


2. 跑道数据
前端方法：
vspAPI.runways(airportId?: string)

后端接口：
GET /api/v1/vsp/runways
GET /api/v1/vsp/runways?airport_id=airport_vhhh

返回类型：
VspRunway[]


3. 通信频率数据
前端方法：
vspAPI.frequencies(airportId?: string)

后端接口：
GET /api/v1/vsp/frequencies
GET /api/v1/vsp/frequencies?airport_id=airport_vhhh

返回类型：
VspFrequency[]


4. Navaid 数据
前端方法：
vspAPI.navaids(airportId?: string)

后端接口：
GET /api/v1/vsp/navaids
GET /api/v1/vsp/navaids?airport_id=airport_vhhh

返回类型：
VspNavaid[]


5. Waypoint 数据
前端方法：
vspAPI.waypoints(keyword?: string, pageSize = 500)

后端接口：
GET /api/v1/vsp/waypoints?page=1&page_size=500
GET /api/v1/vsp/waypoints?keyword=ITFC&page=1&page_size=500

返回类型：
PageResult<VspWaypoint>


6. 程序数据
前端方法：
vspAPI.procedures(airportId?: string)

后端接口：
GET /api/v1/vsp/procedures
GET /api/v1/vsp/procedures?airport_id=airport_vhhh

返回类型：
VspProcedure[]

说明：
如果后端返回 path_geojson，前端地图会尝试绘制程序路径。


7. 航司数据
前端方法：
vspAPI.airlines(keyword?: string)

后端接口：
GET /api/v1/vsp/airlines
GET /api/v1/vsp/airlines?keyword=CX

返回类型：
VspAirline[]

三、前端展示位置

1. 右侧 VSP/AIP 辅助信息面板
文件：
front/src/components/auxiliary-info.tsx

展示内容：
Airports / Runways
Frequencies / Navaids / Waypoints
SID / STAR
Airlines / Callsigns

支持按关键字搜索，例如：
CX
CATHAY
119.1
ITFC
07L


2. 主页面 VSP/AIP 数据状态条
文件：
front/src/components/annotation-page.tsx

展示内容：
VSP/AIP data loaded
Airports 数量
Runways 数量
Frequencies 数量
Navaids 数量
Waypoints 数量
Procedures 数量
Airlines 数量


3. 地图 VSP/AIP 图层
文件：
front/src/components/adsb-map-leaflet.tsx

地图可视化内容：
橙色线：Runway
紫色点/标签：Navaid
蓝色点：Waypoint
绿色点：Airport point
程序路径：如果后端 procedure.path_geojson 有数据，则绘制为航路线


四、前端请求代理配置

文件：
front/next.config.ts

当前前端请求：
/api/v1/:path*

会代理到：
http://127.0.0.1:8000/api/v1/:path*

因此前端不需要手动拼接完整后端地址，默认可以通过 Next.js rewrite 访问 Alpha 后端。

七、说明

本次前端改动的核心目标是：让前端不再使用 mock VSP/AIP 数据，而是读取 Alpha 后端已经导入数据库的真实 VSP/AIP 数据，并在右侧面板和地图中展示。

目前 VSP/AIP 已覆盖机场、跑道、频率、Navaid、Waypoint、程序和航司数据。

后续可继续完善的部分：
1. 后端导入脚本继续解析真实 SID / STAR / approach 程序数据。
2. 地图图层增加点击详情弹窗。
3. 系统日志、队列、死信、用户管理等 Alpha 管理类数据可单独做后台管理页面。
