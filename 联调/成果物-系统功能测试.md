## 5. 缺陷记录与修复（兼容性重点）

| Bug ID | 分类 | 现象 | 根因 | 修复措施 | 状态 |
|--------|------|------|------|----------|------|
| **BUG-001** | 端口冲突 | A5 与 A1 无法同占 8000 | 两模块默认同端口 | A5 独占 8000；A1 改 sync 导入 | **已修复** |
| **BUG-002** | CORS | `Failed to fetch`、OPTIONS 失败 | 未配 CORS；GET 带 JSON Content-Type | `main.py` 加 CORSMiddleware；`requestJson` 仅 POST 设 Content-Type | **已修复** |
| **BUG-003** | 媒体 URL | WaveSurfer 404 `localhost:3000/temp/...` | 相对 `source_url` 解析到前端源 | `resolveBrowserAudioUrl()`；联调数据改绝对 URL | **已修复** |
| **BUG-004** | 构建/加载 | ChunkLoadError layout 超时 | webpack 冷编译 >20s | Turbopack；删 `.next`；合并 AppProviders | **已修复** |
| **BUG-005** | 数据一致 | 列表 7 条 vs A2 库 2 条 | 重复 sync + 测试行 | `file_name` 去重；文档标明真相源 | **已修复** |
| **BUG-006** | 媒体 | en.wav 请求 A3 404 | A3 未挂 StaticFiles / 路径错误 | A3 mount `/media`；副本放 A5 `/static/audio/` | **已修复** |
| **BUG-007** | 性能 | 3343 点地图卡顿 | Leaflet 全量渲染 | 文档标注已知局限；bbox 查询待迭代 | **已知/待办** |
| **BUG-008** | 环境 | PowerShell `&&` 报错 | shell 语法差异 | 文档改用 `;`；脚本用绝对 Path | **已修复** |
| **BUG-009** | 网络 | `127.0.0.1:3000` 偶发超时 | Next 监听 IPv6 localhost | 文档推荐 `localhost:3000` | **已规避** |

## 7. 测试结论

1. **功能**：在「A5 有数据 + A2 媒体可用 + 已执行 sync」前提下，系统完成 **登录—列表—播放—文本轴—地图—导出** 最小闭环，**11 项主用例 + 3 轮回归全部通过**。  
2. **兼容性**：CORS、媒体 URL、端口冲突、Chunk 加载、Shell 语法等 **9 项兼容性问题中 8 项已修复或规避**，1 项（大地图性能）已文档化为已知局限。  
3. **数据**：A5 与 A2 条数不一致属 **重复入库**，非前端逻辑缺陷，已通过 sync 去重与联调文档闭环。  
4. **建议**：后续补充 Playwright E2E、航迹 bbox 分页、时间轴删除 ext 持久化。
