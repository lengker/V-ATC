# Alpha A-5 模块对接矩阵

## 1. 目标

本矩阵用于明确：

- 谁生产数据
- 谁消费数据
- 谁拥有字段定义权
- A-5 在链路中的职责

## 2. 对接矩阵

| 链路 | 生产方 | 消费方 | 字段定义权 | A-5 职责 | 当前结论 |
| --- | --- | --- | --- | --- | --- |
| 用户登录与鉴权 | A-5 | A-4 / 管理端 | A-5 | 直接实现 | 可直接编码 |
| 用户管理 | A-5 | A-4 / 管理端 | A-5 | 直接实现 | 可直接编码 |
| VSP/AIP 数据查询 | A-5 | A-4 | A-5 | 直接实现 | 可直接编码 |
| 队列监控与失败记录 | A-5 | A-5 / 管理端 | A-5 | 直接实现 | 可直接编码 |
| ADS-B 航迹接入 | A-1 | A-5 | A-1 主导，A-5 兼容 | 接口、入库、索引 | 先做最小兼容 |
| 语音元数据接入 | A-2 | A-5 | A-2 主导，A-5 兼容 | 接口、入库、查询 | 先做最小兼容 |
| ASR 结果接入 | A-3 | A-5 | A-3 主导，A-5 兼容 | 接口、入库、版本兼容 | 先做最小兼容 |

| 标注通知 | A-5 | A-4 | A-5 主导外壳 | 消息通知 | 可先定义事件壳 |

## 3. 可直接落地的部分

- 用户表与鉴权接口
- VSP 表与查询接口
- 死信/失败/日志表
- Redis 队列命名和消息外壳

## 4. 必须保留协商空间的部分

- `adsb_tracks` 详细字段
- `a2_voice_info` 扩展字段
- `asr_results.vad_segments_json` 的细粒度格式
- `annotation_tasks.status` 的完整状态流
- `annotation_results.annotations_json` 的细粒度结构

## 5. 实施建议

后续进入编码时，优先级建议为：

1. 主控域先实现
2. 协商型接口先按最小字段实现 DTO
3. 业务字段扩展通过 `raw_payload` / `extra_json` 演进
