# A-2 语音处理模块

本项目是可视化 ATC 地空通话语音标注系统中的 A-2 语音处理模块后端实现。

当前版本已经完成 A-2 的核心能力落地，包括：

- 实时流任务配置、ASX 入口解析与接收落库
- 历史下载任务配置与 LiveATC 归档文件接入
- 语音分段文件入库与元信息管理
- 本地规范化存储
- 按时间范围查询语音
- 跨分段任意时段语音截取与拼接
- 元信息后台同步
- 单元测试、接口测试与集成测试

## 当前状态

目前这套代码已经具备课程项目演示所需的主体能力，尤其是下面这条核心链路已经打通：

1. 语音按分段方式存储到数据库和文件系统
2. 按时间范围查询命中重叠分段
3. 根据目标时段对多个分段做裁剪
4. 将裁剪后的片段按顺序拼接成完整语音

同时，实时流接收、历史下载、本地落盘、按时间查询与文件回传这几条核心链路已经通过集成测试验证。

也就是说，A-2 已经支持“底层分段存储，上层任意时段截取”。

## 已实现功能

### 1. 任务配置

- 支持实时任务创建、查询、连接测试
- 支持通过 `.asx` 入口文件创建 LiveATC 实时接收任务
- 支持实时接收任务启动、停止、状态查询
- 支持历史下载任务创建、查询
- 支持 LiveATC 归档文件 URL 下载与本地文件导入
- 历史 LiveATC 归档音频最多按 30 分钟入库，超出部分仅截取前 30 分钟
- 支持历史下载执行进度更新

### 2. 语音存储与元信息管理

- 按 `机场 ICAO / 频段 / 日期 / 文件名` 进行本地目录存储
- 元信息写入 `a2_voice_info`
- 支持语音文件路径、大小、时间范围、数据类型等字段管理
- 支持落盘后通过后台同步修正 `file_size / checksum / valid_status`
- 实时流按系统接收时间生成 `start_at / end_at / original_time`
- 历史归档文件按文件名开始时间生成起始时间，结束时间最多延伸 30 分钟；当音频超过 30 分钟时仅保留前 30 分钟片段入库

### 3. 语音查询与时段截取

- 支持按 `startTime + endTime + ICAO + band + pageNum + pageSize` 查询
- 已提供 `GET|POST /api/a2/voice/query` 服务接口，返回语音元信息、关联航迹标识和文件下载地址
- 已提供 `GET /api/a2/voice/export` 直接按起止时间导出音频，适合联调和演示
- 查询逻辑使用时间重叠匹配：
  - `segment.start_at < endTime`
  - `segment.end_at > startTime`
- 支持跨多个分段裁剪并拼接完整音频
- `wav` 文件可直接拼接
- `mp3` 或混合格式需要本机安装 `ffmpeg`

### 4. 可靠性相关能力

- 历史下载支持 `.part` 临时文件
- 支持基于 HTTP `Range` 的简化断点续传
- 实时连接支持心跳检测
- 支持 `10s -> 30s -> 60s` 自动重连退避
- 元信息支持后台定时同步
- 元信息支持手动触发同步
- 上传临时文件与导出切片会在接口完成后自动清理
- 当数据库记录存在但物理文件丢失时，文件下载接口会返回明确的 `404`

### 5. 测试

- 已补充单元测试
- 已补充接口测试
- 已补充集成测试，覆盖实时流接收、历史下载、本地存储、按时间查询与元信息同步修复链路
- 当前共 21 个测试，已全部通过

## 运行方式

```bash
pip install -r requirements.txt
python scripts/init_db.py
uvicorn app.main:app --reload
```

SQLite 数据库默认生成在：

```bash
storage/a2.sqlite3
```

## 当前剩余问题

除了已经跑通的主体功能外，当前还存在以下问题或不足：

1. 时间戳语义需要明确  
实时流时间戳是系统接收与分段时间戳，不是上游源侧原生时间戳；历史归档时间戳由文件名开始时间确定起点，结束时间最多取前 30 分钟，对超过 30 分钟的归档音频只保留前 30 分钟片段。

2. 调度能力还是简化版  
断点续传、限速已经有基础实现，但还没有完整的任务调度器；优先级字段已设计，但没有真正的资源调度。

3. 元信息同步还不是完整自动修复版  
现在能扫描、能标记缺失、能修正大小和校验值，但不会自动把数据库外的孤儿文件完整重建成元信息记录。

4. 测试覆盖还可以继续扩展  
目前已经覆盖核心业务与主要接口，但实时监控异常路径、下载失败分支、文件下载接口等还可以继续补测。

5. 课程交付层面的材料还可以继续完善  
当前仓库已经补充 `docs/defense.md`，可以直接作为“模块讲解稿 + 演示顺序 + 未完成原因说明”的基础版本继续细化。
