# 快速启动指南

## 前置要求

- Node.js 18.x 或更高版本
- npm 或 yarn

## 安装步骤

1. **安装依赖**
   ```bash
   npm install
   ```

2. **配置环境变量**
   
   创建 `.env.local` 文件：
   ```env
   NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
   ```
   
   将 `http://localhost:8000` 替换为你的后端 API 地址。

3. **启动开发服务器**
   ```bash
   npm run dev
   ```

4. **访问应用**
   
   打开浏览器访问：http://localhost:3000
   
   如果需要加载特定音频，使用 URL 参数：
   ```
   http://localhost:3000?audioId=your-audio-id
   ```

## 功能说明

### 主要功能区域

1. **左侧面板**
   - 音频波形可视化
   - 播放控制（播放/暂停、进度条、音量）
   - 时间戳列表

2. **中间面板**
   - ADSB 航迹地图
   - 文本编辑器（编辑时间戳）

3. **右侧面板**
   - 音频信息
   - 飞机信息

### 操作说明

- **播放音频**：点击播放按钮或拖动进度条
- **查看航迹**：地图自动显示当前时间点的飞机位置
- **选择飞机**：点击地图上的飞机标记
- **编辑时间戳**：点击时间戳列表中的"编辑"按钮
- **时间同步**：所有组件会根据当前播放时间自动同步

## 开发模式

开发模式下，代码会自动热重载。修改代码后，浏览器会自动刷新。

## 构建生产版本

```bash
npm run build
npm start
```

## 故障排除

1. **地图不显示**
   - 确保网络连接正常（需要加载 OpenStreetMap 瓦片）
   - 检查浏览器控制台是否有错误

2. **音频无法播放**
   - 检查音频 URL 是否可访问
   - 确保浏览器支持 Web Audio API

3. **API 请求失败**
   - 检查 `.env.local` 中的 API 地址是否正确
   - 确保后端服务正在运行
   - 检查浏览器控制台的网络请求

## 与后端对接

前端通过以下 API 端点与后端（A-5 模块）通信：

- `GET /api/audio/:id` - 获取音频数据
- `GET /api/adsb/:audioId` - 获取 ADSB 数据
- `PUT /api/audio/:id/timestamps` - 更新时间戳

确保后端实现了这些接口，并返回符合 `src/types/index.ts` 中定义的数据格式。
